What is the idea behind the method?

In statistics, bootstrap methods are a class of methods that estimate the distribution of a population by resampling. The observed sample is treated as a population and random samples with replacement are generated (resampled) from it. With bootstrap or bootstrapping it is possible to study the population as from itâs pseudo-population.

Figure: Representation of Bootstrap with 3 âresamplesâ of size 5.

  • The Bootstrap sample is the same size (n = 5) as the original sample (it can also be larger or smaller).

  • Each Boostrap sample is nothing more than a resample, that is, a simple sampling with replacement made from the original sample.

  • The fact of being with replacement means that the just sorted value remain in the set of possible values to the next withdrawal.

  • B is the arbitrary number of replicas (number of resamples) bootstrap.

How and why does it work?

We saw that resampling methods like jackknife and bootstrap are based on resamples from a sample from the target population under study. In the case of Bootstrap, \(\mathbf{x} = (x_1, \dots, x_n)\) is a random sample (r.s.) with observed replacement of an unknown population distribution \(F_X\).

Therefore, let \(X^â\) be the random variable that represents the selection of an observation randomly from \(\mathbf{x}\), then \[p = P (X^â=x_i) = \frac{1}{n}, ~i = 1, \dots, n.\]

Note that \(X^*\) follows a discrete uniform distribution in the set \(\{x_1, \dots, x_n \}\). So, in this case, resampling B times is equivalent to generating B bootstrap samples of size \(m\) with independent and identically distributed variables (i.i.d.) \[\mathbf{X^*} = (X^*_ 1, X^*_2, \dots, X^*_m), \text{ with } X^* \sim Unif \{x_1, \dots, x_n \}.\]

Example 7.1 Rizzo (2007): Suppose Suppose \(\mathbf{x} = {2,2,1,1,5,4,4,3,1,2}\) r.s. observed size 10. Note that, given random sample \(\mathbf{x}\), there are 5 possible values to form a sample \(X^â\) of \(\mathbf{x}\) with probabilities:

\[ P (X = k) = \begin {cases} 0.3; \text{if } k = 1 \\ 0.3; \text{if } k = 2 \\ 0.1; \text{if } k = 3 \\ 0.2; \text{if } k = 4 \\ 0.1; \text{if } k = 5 \\ \end {cases} \]

Also, note that each element belonging to the set \(\{2,2,1,1,5,4,4,3,1,2 \}\) has a probability of withdrawal \(p = 1/10\), so the probabilities are added as the values are repeated in that set.

Visualization in R:

x <- c(2,2,1,1,5,4,4,3,1,2)
n <- length(x)
barplot(table(x)/n, bty = "n", ylab = expression(P(X == k)),
        xlab = expression(k), col = "darkblue")

Note that the accumulated \(F_X^â\) distribution of the bootstrap sample with the empirical distribution that can be used as an approximation of \(F_X\).

Remember that \(F_n\) is a discrete distribution with equal weights for each sample point, \(F_n\) is defined as a sample proportion \(F_n(t) = \sum \limits_{i = 1}^n \# \{x_i \le t \} I_{(x_i\le t)}\).

\[ F_n(k) = F_X^â(k) = \begin{cases} 0; \text{se} k <1 \\ 0.3; \text{se} 1 \le k <2 \\ 0.6; \text{se} 2 \le k <3 \\ 0.7; \text{se} 3 \le k <4 \\ 0.9; \text{se} 4 \le k <5 \\ 1; \le k \ge 5. \\ \end{cases} \]

With the help of R, it is possible to visualize that the two distributions actually coincide, therefore, resampling of \(x\) is equivalent to generating values of the \(F_n (x)\) distribution.

        xlab = expression (x [i]), col = "darkblue")

plot(ecdf (x), main = "Empirical Dist", bty = "n", ylab = expression(F [n] (k)))

Therefore, one of the two following assumptions is made. Either the empirical distribution of bootstrap replicas is a good approximation for \(F_n\), or the \(F_n\) empirical distribution function is a good approximation for \(F_X\), so the distribution of the bootstrap estimator \(\hat \theta^*\) is similar to the unknown distribution of the originally proposed estimator \(\hat \theta\).

Therefore, the task of making inferences about \(\theta\) comes down to learning about the bootstrap distribution of \(\hat \theta^*\). Therefore, for each bootstrap sample indexed by \(b=1, \dots, B\), \(\hat \theta^{*(b)}\) is calculated and the bootstrap estimate of the estimator distribution \(F_{\hat \theta} (.)\) is the empirical distribution of the replicas \(\hat \theta^{* (1)}, \dots, \hat \theta^{*(B)}\).

The sample above is actually a sample of a $Poisson (2) $. A large number of replicas produce a good estimate of $F_n $, but not a good estimate of $F_X $. The violation of the second assumption occurs because regardless of the number of replicates, the bootstrap samples will never include the value zero (Rizzo 2007).

Conclusion

The distribution of the finite population represented by the sample can be considered a pseudo-population with characteristics similar to those of the true population. More than that, we learned that through the repeated generation of random samples from this pseudo-population (resampling), the sample distribution of a statistic can be estimated. Therefore, the bootstrap estimates of a sampling distribution are analogous to the idea of density estimation. Know that there are several advanced applications of bootstrap, the properties of an estimator such as bias or standard error, for example, can be estimated by resampling methods.
